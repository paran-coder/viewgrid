# ViewGrid 사용자 매뉴얼

> 문서 버전: 1.0.0  
> 상태: Stage 6 완료 — 공개 운영 버전  
> 최종 갱신: 2026-07-19

## 1. ViewGrid란 무엇인가요?

ViewGrid는 이미지 한 장 위에 가상 카메라를 배치하고 좌우·상하 각도, 기울기, 화각과 거리를 조절해 새로운 시점의 이미지 세트를 생성하는 브라우저 스튜디오입니다.

ViewGrid는 실제 3D 스캔이나 정확한 360도 복원 도구가 아닙니다. 큰 각도에서 새로 보이는 영역은 이미지 모델이 추론하므로 제품 구조, 로고와 글자가 달라질 수 있습니다.

ViewGrid 1.0.0은 다음 세 종류의 입력을 역할별로 나눕니다.

1. **원본 이미지:** 제품 외형과 세부 구조 기준
2. **로컬 카메라 가이드:** 목표 각도·화각·프레이밍 기준
3. **인접 완성 결과:** 여러 카메라 결과의 재질·구조·조명 일관성 기준

## 2. 준비물

### 브라우저

최신 Chrome, Edge 또는 Safari를 권장합니다. JavaScript와 Canvas가 활성화되어 있어야 합니다.

### 입력 이미지

지원 형식:

- JPEG
- PNG
- WebP

업로드 허용 크기는 최대 12MB입니다. 생성 전 브라우저의 전처리 Worker에서 긴 변과 파일 크기를 줄입니다. Worker를 지원하지 않거나 처리에 실패하면 메인 스레드 Canvas 방식으로 자동 전환됩니다.

권장 이미지:

- 피사체가 하나인 이미지
- 피사체 전체가 잘리지 않은 이미지
- 배경과 피사체가 명확히 구분되는 이미지
- 정면 또는 약한 사선 시점
- 로고와 작은 구조가 선명한 이미지

불안정할 수 있는 입력:

- 투명하거나 거울처럼 반사되는 물체
- 머리카락·식물·그물처럼 경계가 복잡한 피사체
- 피사체가 이미지 가장자리에 붙은 사진
- 여러 물체가 서로 크게 가려진 사진
- 매우 강한 원근이나 초광각 원본
- 작은 글자와 반복 패턴이 많은 제품

### API 키

다음 중 하나가 필요합니다.

- OpenAI API 키
- Google Gemini API 키

API 사용료와 모델 접근 권한은 사용자의 공급자 계정에 적용됩니다.

## 3. API 키 처리 방식

- API 키는 브라우저 메모리에만 보관됩니다.
- `localStorage`, 쿠키, IndexedDB 또는 DB에 저장하지 않습니다.
- 페이지를 새로고침하거나 탭을 닫으면 사라집니다.
- **이 탭이 열려 있는 동안 유지**를 끄면 한 장 또는 전체 큐가 끝난 후 삭제됩니다.
- 키와 이미지는 생성 요청 동안 HTTPS로 Vercel Function과 선택 공급자에게 전달됩니다.
- JSON 프리셋, 세션 설정, ZIP 메타데이터에는 API 키가 포함되지 않습니다.

## 4. 시작하기

1. ViewGrid를 엽니다.
2. 이미지를 선택하거나 업로드 영역에 끌어 놓습니다.
3. 빠른 확인이 필요하면 **데모 이미지로 시작**을 누릅니다.
4. 이미지 위의 C1–C9 카메라 중 하나를 선택합니다.
5. 카메라 수치 또는 기본 프리셋을 조절합니다.
6. **로컬 카메라 가이드**에서 원본과 목표 구도를 확인합니다.
7. **멀티뷰 일관성**에서 참조와 정규화 옵션을 확인합니다.
8. API 공급자와 키를 설정합니다.
9. 한 장 또는 활성 카메라 전체를 생성합니다.
10. 결과의 `GUIDE`, `REF`, `NORM`, `STALE` 상태를 검토합니다.
11. 3×3 PNG 또는 ZIP을 내려받습니다.

## 5. 카메라 파라미터

### Yaw — 좌우 각도

- 안정적: ±25° 이내
- 주의: ±26°–45°
- 실험적: ±46°–60°

Yaw가 커질수록 원본에 보이지 않던 옆면을 모델이 더 많이 추론합니다.

### Pitch — 상하 각도

- 안정적: ±10° 이내
- 주의: ±11°–20°
- 실험적: ±21°–30°

Pitch가 커질수록 윗면 또는 밑면이 새로 드러납니다.

### Roll — 기울기

허용 범위는 ±20°입니다. 로컬 가이드에서는 이미지 평면 회전으로 반영합니다.

### FOV — 화각

- 20°–30°: 망원에 가까운 압축 원근
- 31°–45°: 짧은 망원
- 46°–60°: 표준 렌즈
- 61°–75°: 광각
- 76°–100°: 강한 광각

### Distance — 거리

피사체의 화면 점유율과 여백을 조절합니다. 범위는 0.6×–2.0×입니다.

## 6. 기본 카메라 프리셋

- **안전한 9뷰:** 작은 각도로 원본 보존을 우선합니다.
- **제품 기본 9뷰:** 제품 상세용 균형 프리셋입니다.
- **측면 강조:** 더 큰 Yaw를 사용하며 구조 변화 위험이 큽니다.
- **광고 구도:** 각도보다 거리와 화각 변화를 강조합니다.

프리셋 적용 후 카메라를 직접 수정하면 사용자 정의 상태로 전환됩니다.

## 7. 로컬 카메라 가이드

로컬 가이드는 선택한 카메라의 목표 구도를 Canvas에서 거칠게 시각화한 이미지입니다.

### 이미지 역할

- **Image 1 — 원본:** 피사체 형태, 비율, 색상, 재질, 로고와 작은 구조의 기준
- **Image 2 — 가이드:** 카메라 방향, 피사체 크기, 화각, 거리와 프레이밍의 기준

가이드는 완성 결과가 아닙니다. 블러, 늘어짐, 스트립 경계와 빈 영역이 보일 수 있으며, 프롬프트는 모델이 이런 왜곡을 복사하지 않도록 지시합니다.

### 가이드 사용

- 켜짐: 원본과 가이드를 함께 전송합니다.
- 꺼짐: 원본만 전송하고 카메라 수치는 텍스트로 전달합니다.

가이드 사용 여부를 바꾸면 기존 결과는 현재 생성 조건과 달라져 `STALE`로 표시됩니다.

### 미리보기 표시·숨김

미리보기를 숨겨도 가이드 사용 스위치가 켜져 있으면 생성 요청에는 가이드가 포함됩니다.

### 가이드 실패

Canvas 처리에 실패하면 해당 요청은 원본 단독으로 자동 전환됩니다. 결과에는 `FALLBACK` 경고가 기록됩니다.

## 8. 멀티뷰 일관성 설정

### 인접 결과 참조

켜면 중앙 기준 결과를 먼저 만들고, 이후 생성에 이미 완성된 인접 결과를 추가 참조로 사용합니다.

기본 생성 순서:

```text
C5 → C4 → C6 → C2 → C8 → C1 → C3 → C7 → C9
```

참조 관계:

| 생성 카메라 | 참조 결과 |
| ----------- | --------- |
| C5          | 없음      |
| C4, C6      | C5        |
| C2, C8      | C5        |
| C1, C7      | C4        |
| C3, C9      | C6        |

참조 우선순위는 다음과 같습니다.

1. 원본 외형
2. 로컬 가이드 구도
3. 인접 결과의 구조·재질·조명

인접 결과가 원본과 다르면 모델은 원본을 우선하도록 지시됩니다. 참조가 없거나 압축에 실패하면 원본과 가이드만으로 계속 생성합니다.

### 크기·색상 정규화

켜면 생성 응답을 브라우저에서 다음처럼 정리합니다.

- 1,024×1,024 정사각형으로 통일
- 피사체를 자르지 않는 범위에서 중앙 배치
- 원본 또는 실제 참조 결과에 가까운 제한된 전역 색상 게인
- WebP로 재인코딩

정규화는 제품 마스크를 분리하거나 배경을 새로 만드는 기능이 아닙니다. 결과 시트의 크기와 전체 톤 차이를 완화하는 보수적인 후처리입니다.

### 옵션 변경

인접 참조 또는 정규화 옵션을 변경하면 기존 결과는 `STALE`이 됩니다. 같은 설정으로 다시 생성해야 현재 3×3 시트에 포함할 수 있습니다.

### 로고와 문자

참조를 사용해도 큰 각도에서는 로고와 문자가 변형될 수 있습니다. 결과를 확대해 글자, 버튼, 손잡이와 작은 연결부를 직접 확인합니다.

## 9. API 공급자 연결

1. **API 설정**을 엽니다.
2. OpenAI 또는 Google Gemini를 선택합니다.
3. 모델을 선택합니다.
4. API 키를 입력합니다.
5. 필요하면 **연결 확인**을 누릅니다.
6. 설정창을 닫습니다.

지원 모델:

- OpenAI `gpt-image-2`
- Gemini `gemini-3.1-flash-image`
- Gemini `gemini-3-pro-image`

## 10. 한 장 생성

1. 생성할 카메라를 선택합니다.
2. 가이드·참조·정규화 설정을 확인합니다.
3. **C번호 한 장 생성**을 누릅니다.
4. 해당 카메라의 선행 참조가 현재 결과에 있으면 함께 사용됩니다.
5. 완료 후 확대·다운로드·재생성·삭제할 수 있습니다.

한 장 생성에서도 참조 정책은 동일하지만, 필요한 선행 결과가 없으면 `NO REF` 상태로 생성합니다.

## 11. 활성 카메라 전체 생성

1. 필요 없는 카메라는 비활성화합니다.
2. 로컬 가이드 사용 여부를 결정합니다.
3. 인접 결과 참조와 정규화 여부를 확인합니다.
4. **활성 N개 실제 생성**을 누릅니다.
5. 원본은 한 번만 전처리됩니다.
6. 활성 카메라는 중앙 기준 계획에 따라 순차 처리됩니다.
7. 화면에서 현재 카메라와 참조 카메라를 확인합니다.
8. 각 결과는 필요하면 1,024px로 정규화됩니다.

동시 실행 수는 1개입니다. 공급자 속도 제한, 브라우저 메모리와 예상치 못한 비용 증가를 줄이기 위한 정책입니다.

## 12. 큐 중단과 재시도

### 큐 중단

생성 중 **큐 중단**을 누르면:

- 이미 완료된 결과는 유지됩니다.
- 현재 요청은 중단됩니다.
- 아직 시작하지 않은 셀은 중단 상태가 됩니다.

### 실패·중단 재시도

**실패·중단 N개 재시도**는 해당 셀만 새 계획으로 처리합니다. 재시도 시 현재 설정과 일치하는 선행 결과만 참조합니다.

재시도 가능한 오류는 해당 셀만 실패로 남기고 다음 카메라를 계속 처리합니다. API 키·권한처럼 복구할 수 없는 오류는 남은 큐를 중단합니다.

## 13. 결과 배지 읽기

### 가이드 배지

- `GUIDE`: 원본과 가이드가 사용됨
- `SOURCE`: 사용자가 가이드를 끄고 원본만 사용함
- `FALLBACK`: 가이드 생성 실패로 원본만 사용함

### 참조 배지

- `REF C5`처럼 표시: 해당 완성 결과가 추가 참조로 사용됨
- `NO REF`: 참조가 꺼졌거나 사용할 수 있는 선행 결과가 없음

### 정규화 배지

- `NORM`: 1,024px 크기와 제한된 색상 게인이 적용됨
- `RAW`: 정규화를 끄거나 후처리 폴백으로 공급자 결과를 유지함

### STALE

다음 조건이 바뀌면 기존 결과가 `STALE`로 표시됩니다.

- Yaw, Pitch, Roll, FOV, Distance
- 카메라 활성 상태
- 공급자 또는 모델
- 가이드 사용 여부
- 인접 결과 참조 사용 여부
- 정규화 사용 여부

`STALE` 결과는 ZIP에 보관할 수 있지만 최신 3×3 완료 조건에는 포함되지 않습니다.

## 14. JSON 프리셋 저장과 불러오기

### 저장

**JSON 저장**을 누르면 다음 설정이 파일로 저장됩니다.

- 카메라 9개의 수치와 활성 상태
- 선택된 카메라
- 가이드 사용 여부
- 인접 참조 사용 여부
- 정규화 사용 여부

저장되지 않는 항목:

- API 키
- 원본·가이드·참조·결과 이미지
- Blob URL
- 공급자 응답 본문

### 불러오기

1. **JSON 불러오기**를 누릅니다.
2. ViewGrid 프리셋 파일을 선택합니다.
3. 스키마와 카메라 ID를 검증합니다.
4. 범위를 벗어난 수치는 현재 하드 제한으로 조정합니다.
5. 설정이 적용되면 기존 결과는 필요에 따라 `STALE`로 판정됩니다.

200KB를 초과하거나 잘못된 JSON 파일은 불러오지 않습니다.

## 15. 탭 단위 설정 복구

ViewGrid는 카메라·가이드·일관성 설정만 `sessionStorage`에 임시 저장합니다.

- 같은 탭에서 페이지를 다시 열면 설정을 복구할 수 있습니다.
- 탭을 닫으면 세션 설정은 삭제됩니다.
- API 키와 이미지는 세션 설정에 포함되지 않습니다.
- 브라우저 정책 또는 저장 공간 제한으로 복구에 실패해도 앱은 기본 설정으로 계속 실행됩니다.

## 16. 결과 검토

각 결과를 확대해 다음을 확인합니다.

- 요청한 카메라 방향과 비슷한가
- 가이드의 피사체 크기와 프레이밍을 따르는가
- 원본 형태와 비율이 유지됐는가
- 인접 결과와 재질·색상·조명이 비슷한가
- 로고와 글자가 깨지지 않았는가
- 버튼·손잡이·연결부가 추가되거나 사라지지 않았는가
- 정규화로 제품색이 과도하게 달라지지 않았는가
- 배경과 그림자가 자연스러운가

참조 결과 자체에 오류가 있으면 후속 카메라에도 영향을 줄 수 있습니다. 기준이 되는 C5, C4, C6을 먼저 검토하고 문제가 있으면 해당 셀을 재생성한 뒤 후속 셀도 다시 생성하는 것이 좋습니다.

## 17. 3×3 PNG 다운로드

모든 활성 카메라에 현재 공급자·모델·카메라·가이드·참조·정규화 설정과 일치하는 결과가 있으면 **3×3 PNG** 버튼이 활성화됩니다.

기본 출력:

- 셀 크기: 768×768px
- 셀 간격: 12px
- 전체 크기: 2,328×2,328px
- 배경: 어두운 중립색
- 라벨: 카메라 번호, Yaw, Pitch, FOV
- 비활성 카메라: `OFF` 자리표시자

시트 조립은 브라우저 Canvas에서 수행되며 서버에 저장하지 않습니다.

## 18. ZIP 다운로드

실제 결과가 한 장 이상 있으면 **결과 ZIP**을 사용할 수 있습니다.

```text
images/
  viewgrid-c1-....webp
  viewgrid-c2-....jpg
  ...
viewgrid-contact-sheet.png  # 모든 활성 결과가 최신인 경우
viewgrid-manifest.json
```

Manifest에는 다음 정보가 포함됩니다.

- 카메라 수치와 활성 상태
- 공급자와 모델
- 가이드 요청·사용·폴백 상태
- 참조 요청·사용·참조 카메라·경고
- 생성 계획 버전
- 정규화 요청·적용·버전·출력 크기·색상 게인
- 처리시간과 생성 시점

API 키와 Blob URL은 포함하지 않습니다.

## 19. 오류 해결

### API 키가 필요하다는 메시지

API 설정을 열어 키를 입력합니다. 유지 옵션을 끈 경우 이전 큐가 끝난 뒤 삭제됐을 수 있습니다.

### 인증 또는 권한 오류

- 선택 공급자에 맞는 키인지 확인합니다.
- 선택 모델 접근 권한을 확인합니다.
- 결제와 사용 한도를 확인합니다.

### 사용량 또는 속도 제한

완료 결과는 유지됩니다. 잠시 후 실패한 셀만 재시도합니다.

### 이미지가 너무 큼

더 작은 원본을 사용하거나 JPEG·PNG·WebP로 다시 저장합니다.

### 참조가 사용되지 않음

- 인접 결과 참조가 켜져 있는지 확인합니다.
- 필요한 선행 결과가 완료·최신 상태인지 확인합니다.
- C5 또는 같은 열 기준 결과가 삭제·실패·STALE인지 확인합니다.
- 참조 압축 경고가 있으면 더 작은 결과 또는 원본으로 다시 시도합니다.

### 기준 결과가 좋지 않음

C5를 먼저 재생성하고, C4·C6을 재생성한 뒤 해당 열의 위·아래 결과를 다시 생성합니다. 오류가 있는 기준 결과를 그대로 두면 후속 결과에 영향을 줄 수 있습니다.

### 색상이 과하게 보정됨

- 정규화를 끄고 한 장을 비교 생성합니다.
- 반사·조명 변화가 큰 원본은 전역 평균색 정규화와 맞지 않을 수 있습니다.
- 원본과 가까운 중앙 기준 결과를 먼저 개선합니다.

### 가이드가 과하게 늘어짐

- Yaw와 Pitch를 안정적 범위로 줄입니다.
- FOV를 45°–60°로 맞춥니다.
- Distance를 0.8×–1.3× 범위로 조정합니다.
- 필요하면 가이드 사용을 끄고 비교합니다.

### 결과가 요청 각도와 다름

- 가이드 사용이 켜져 있는지 확인합니다.
- Yaw와 Pitch를 안정적 범위로 줄입니다.
- 정면에 가까운 원본을 사용합니다.
- 해당 셀을 다시 생성합니다.

### 3×3 버튼이 비활성

- 실패·중단 셀이 있는지 확인합니다.
- `STALE` 셀이 있는지 확인합니다.
- 공급자나 모델을 바꿨는지 확인합니다.
- 현재 가이드·참조·정규화 모드와 다른 결과가 있는지 확인합니다.
- 비활성 카메라는 결과가 없어도 됩니다.

## 20. 세션 종료 전 주의사항

- 페이지를 닫기 전에 결과를 내려받습니다.
- 새로고침하면 API 키와 생성 결과는 사라질 수 있습니다.
- 카메라 설정은 같은 탭에서 임시 복구될 수 있지만 영구 프로젝트 저장이 아닙니다.
- 재사용할 설정은 JSON으로 내려받습니다.
- 공용 컴퓨터에서는 API 키를 삭제하고 탭을 닫습니다.

## 21. PWA 설치와 오프라인 동작

지원 브라우저에서는 상단의 설치 아이콘 또는 브라우저 메뉴를 이용해 ViewGrid를 앱처럼 설치할 수 있습니다.

오프라인에서 가능한 작업:

- 이미 열린 앱 셸 다시 열기
- 카메라 설정 조정
- 현재 메모리에 남아 있는 결과 확인·다운로드
- JSON 프리셋 저장

오프라인에서 불가능한 작업:

- API 키 연결 확인
- OpenAI·Gemini 실제 생성
- 공급자 오류 또는 사용량 확인

Service Worker는 앱 셸과 정적 자산만 캐시합니다. `/api/` 요청, API 응답, 사용자 이미지와 생성 Blob은 캐시하지 않습니다.

## 22. 보안과 개인정보 확인

- API 키는 현재 탭 메모리에만 둡니다.
- 서버 응답과 생성 이미지는 캐시 금지로 전달됩니다.
- 요청마다 ViewGrid 요청 ID가 붙을 수 있지만 API 키와 이미지 내용은 포함하지 않습니다.
- 공개 배포는 Vercel Firewall 속도 제한을 함께 사용합니다.
- 선택한 OpenAI 또는 Google은 각자의 약관과 데이터 처리 조건에 따라 요청을 처리합니다.
- 공용 기기에서는 키 삭제 후 탭을 닫고, 내려받은 파일도 직접 삭제합니다.

자세한 내용은 저장소의 `PRIVACY.md`와 `SECURITY.md`를 확인합니다.

## 23. 키보드와 화면 낭독기 사용

- 페이지 첫 포커스에서 **본문으로 건너뛰기** 링크를 사용할 수 있습니다.
- Tab과 Shift+Tab으로 버튼, 카메라, 입력과 다운로드 사이를 이동합니다.
- 슬라이더는 화살표로 1단계, Shift+화살표로 5단계 조정합니다.
- 확대 모달과 API 설정 모달은 Escape로 닫습니다.
- 모달이 열린 동안 Tab 포커스는 모달 안에서 순환합니다.
- 생성 진행률, 연결 상태, 오류와 다운로드 완료 메시지는 상태 영역으로 전달됩니다.
- 운영체제에서 동작 줄이기를 켜면 회전·전환·스피너 움직임이 최소화됩니다.

## 24. 운영 상태 확인

공개 배포의 앱 자체 상태는 `/api/health`에서 확인할 수 있습니다. 이 주소는 OpenAI 또는 Gemini를 호출하지 않으므로 공급자 상태까지 보장하지는 않습니다.

지속적인 401·403은 키 또는 모델 권한, 429는 공급자·Vercel 속도 제한, 502·504는 공급자 통신 또는 시간 초과를 우선 확인합니다.
