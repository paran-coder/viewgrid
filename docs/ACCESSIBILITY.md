# 접근성 점검 기록

## 적용 항목

- 문서 언어 `ko`와 명확한 제목·설명
- 본문 바로가기 링크
- 키보드로 업로드, 카메라 선택, 슬라이더, 프리셋, 생성과 다운로드 사용
- 모든 핵심 포커스 대상의 고대비 포커스 링
- 슬라이더 레이블, 도움말 연결과 현재 값 텍스트
- 상태를 색상 외 텍스트·아이콘·배지로 중복 표현
- 생성 진행률 `progressbar`와 `aria-busy`
- 연결·오류·내보내기 상태 live region
- API 설정과 확대 모달의 Escape, Tab 순환과 포커스 복귀
- 44px 터치 대상 보완
- `prefers-reduced-motion`에서 애니메이션과 전환 최소화
- 360px, 768px, 1440px Playwright 시나리오
- axe serious·critical 위반 차단

## 수동 확인 항목

배포 환경에서 다음을 반복합니다.

1. 키보드만으로 업로드부터 ZIP 다운로드까지 이동합니다.
2. NVDA 또는 VoiceOver로 카메라와 슬라이더 이름·값을 확인합니다.
3. 확대 모달을 열고 Tab이 모달 밖으로 이동하지 않는지 확인합니다.
4. 200% 확대와 320 CSS px 폭에서 가로 스크롤과 잘림을 확인합니다.
5. 강제 색상 모드와 축소 모션에서 상태 식별이 가능한지 확인합니다.
6. 실제 생성 오류가 포커스를 잃지 않고 낭독되는지 확인합니다.
