## 2. 디자인 점검

- 구조를 스타일보다 먼저 분리했다.
- 캔버스, 설정 패널, 프리셋, 생성 행동의 계층이 명확하다.
- 핵심 CTA에만 노란색 액센트를 사용했다.
- 과한 블러와 글로우를 사용하지 않았다.
- 상태는 색상뿐 아니라 아이콘과 텍스트로 표시한다.
- 모든 카메라 수치는 tabular numerals를 사용한다.
- Pretendard CDN과 시스템 폴백을 적용했다.
- 모바일에서는 단일 컬럼, 데스크톱에서는 캔버스·패널 분할 구조다.

현재 실행 환경의 시스템 Chromium에는 `URLBlocklist: ["*"]` 관리 정책이 적용되어 모든 URL 접근이 차단됐다. 이 때문에 로컬 브라우저 E2E와 스크린샷 실행은 완료하지 못했다. 제품 코드 오류가 아니라 점검 환경 제약이며, GitHub Actions에서는 `npx playwright install --with-deps chromium` 후 E2E를 실행하도록 구성했다.

## 4. 점검 중 발견하고 수정한 사항

### 업로드 영역 중첩 인터랙션

초기에는 클릭 가능한 업로드 컨테이너 안에 별도의 데모 버튼이 포함돼 중첩 인터랙션 문제가 있었다.

수정:

- 드롭 영역은 비인터랙티브 컨테이너로 변경
- 이미지 선택과 데모 시작을 독립 버튼으로 분리
- 키보드 포커스 순서를 단순화

### 프리셋 선택 상태

카메라 수동 수정 후에도 제품 기본 프리셋이 선택된 것처럼 표시될 수 있었다.

수정:

- 프리셋 선택 상태에 `custom`을 추가
- 수동 파라미터 변경 시 프리셋 활성 표시 해제

### 테스트 범위 오염

Vitest가 Playwright E2E 파일까지 수집해 테스트 러너가 충돌했다.

수정:

- Vitest 수집 범위를 `src/**/*.test.*`로 제한
- E2E와 빌드 결과 폴더를 테스트·린트 대상에서 제외

### 라이브러리 아이콘 이름 변경

현재 Lucide 패키지의 내보내기 이름과 초기 코드가 일치하지 않았다.

수정:

- 실제 패키지 내보내기를 확인해 `GitBranch`, `Grid3X3`으로 교체
