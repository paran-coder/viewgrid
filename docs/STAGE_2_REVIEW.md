# Stage 2 자체 점검 보고서

> 평가일: 2026-07-19  
> 대상 버전: 0.2.0  
> 단계: 사용자 API 키 기반 단일 실제 생성

## 1. 단계 목표

Stage 2의 목표는 Stage 1 카메라 편집기에서 선택한 카메라 한 장을 사용자의 OpenAI 또는 Google Gemini API 키로 실제 생성할 수 있도록 만드는 것이었다.

핵심 완료 조건:

- 공급자·모델·API 키 설정
- 키를 영구 저장하지 않는 Vercel Function 중계
- OpenAI·Gemini 공급자 어댑터
- 카메라 수치 기반 프롬프트
- 입력 이미지 전처리
- 한 카메라 생성·취소·오류·재시도·다운로드
- 테스트와 문서 갱신

- API 키 표시·숨김

- JPEG 품질 반복 조절로 2.2MB 이하 압축
- 카메라 수치를 촬영 언어로 변환
- 선택한 카메라 한 장 생성
- 로딩·취소·오류·재시도
- 확대 모달
- MIME 기반 다운로드 확장자

## 3. 자체 점검에서 발견하고 수정한 문제

### 3.1 런타임별 File 판별

문제: `instanceof File`은 Node·jsdom·브라우저의 전역 객체 차이로 검증이 흔들릴 수 있었다.

수정: FormData 값이 문자열인지 여부를 기준으로 업로드 파일을 판별하고 MIME·크기를 별도로 검증했다.

### 3.2 공급자 라디오 접근성 이름

문제: 설명문까지 라디오의 접근성 이름에 포함되어 테스트와 실제 화면 낭독기 탐색이 불필요하게 길어졌다.

수정: 명시적인 레이블과 역할 기반 선택 구조로 정리하고 접근성 테스트를 갱신했다.

### 3.3 Vercel 응답 크기 위험

문제: 1K PNG가 함수 응답 제한에 근접하거나 초과할 수 있었다.

수정: OpenAI는 WebP 품질 85, Gemini는 1K JPEG를 우선 요청하며 서버에서 4.2MB 출력 상한을 다시 검사한다.

### 3.4 잘못된 다운로드 확장자

문제: 공급자 출력이 WebP 또는 JPEG여도 파일명이 항상 PNG가 될 수 있었다.

수정: 실제 `Content-Type`으로 확장자를 결정한다.

### 3.5 잘못된 JSON 오류

문제: 연결 확인 요청 본문이 잘못된 JSON이면 내부 오류로 처리될 가능성이 있었다.

수정: 파싱 실패를 명시적인 400 `invalid_request`로 반환한다.

### 3.6 취소와 타임아웃 구분

문제: 사용자 취소와 공급자 장시간 응답이 같은 오류로 보일 수 있었다.

수정: 요청 취소는 `request_cancelled`, 240초 초과는 `generation_timeout`으로 구분하고 재시도 가능 상태로 표시한다.

### 3.7 네트워크 오류 정규화

문제: `fetch` 네트워크 실패가 내부 오류처럼 보일 수 있었다.

수정: `provider_network_error`로 변환하고 공급자 원문이나 키가 응답에 포함되지 않도록 했다.

### 3.8 키 삭제와 모달 포커스

문제: 사용자가 메모리에서 키를 즉시 지우는 명시적 액션과 모달 포커스 순환이 필요했다.

수정: 키 삭제 버튼, Escape 닫기, 열기 버튼으로 포커스 복귀, Tab·Shift+Tab 순환을 추가했다.

## 5. 보안·개인정보 점검

- API 키를 환경변수, 저장소, 쿠키, IndexedDB와 DB에 저장하지 않음
- 키를 요청 헤더에만 전달
- 키와 이미지를 로그로 출력하는 코드 없음
- 공급자 원문 오류를 그대로 반사하지 않음
- 응답 캐시 금지
- MIME·파일 크기·프롬프트 길이·허용 모델 검증
- 클라이언트 요청 취소를 공급자 요청까지 전달
- Blob URL을 이미지 교체·프로젝트 초기화 시 해제
- 현재 탭 유지 해제 시 요청 후 키 삭제

남은 점검:

- Vercel 배포 로그에서 민감정보 비노출 확인
- 실제 공급자 요청 ID와 오류 케이스 확인
- 보안 헤더 전체 정책은 Stage 6에서 강화

## 6. 남은 위험

1. 실제 공급자 API 응답 구조가 계정, 리전 또는 출시 상태에 따라 달라질 수 있다.
2. Vercel 4.5MB 한도는 압축으로 완화했지만 복잡한 결과는 여전히 초과할 수 있다.
3. 사용자의 API 키가 Vercel Function을 통과하므로 서비스 문구가 이를 정확히 알려야 한다.
4. 이미지 모델은 카메라 수치를 실제 3D 각도로 보장하지 않는다.
5. EXIF 방향별 실기기 fixture 테스트가 남아 있다.
6. 브라우저 E2E는 CI와 실제 배포 환경에서 다시 확인해야 한다.
7. 9개 실제 생성 큐와 3×3 다운로드는 Stage 3 범위다.
