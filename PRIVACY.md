# ViewGrid 개인정보 처리 안내

> 적용 버전: ViewGrid 1.0.0  
> 최종 갱신: 2026-07-19

## 처리하는 정보

ViewGrid 애플리케이션은 다음 데이터를 기능 수행 중 일시적으로 처리합니다.

- 사용자가 선택한 원본 이미지
- 브라우저가 만든 카메라 가이드
- 앞서 생성된 인접 참조 이미지
- 사용자가 입력한 OpenAI 또는 Google Gemini API 키
- 공급자·모델·카메라 설정
- 생성된 이미지와 내보내기 메타데이터

## 데이터 흐름

1. 원본은 브라우저에서 리사이즈·압축됩니다.
2. 생성 시 API 키와 필요한 이미지만 동일 출처 Vercel Function으로 전송됩니다.
3. Function은 선택한 OpenAI 또는 Google API에 요청을 중계합니다.
4. 결과는 캐시 금지 응답으로 브라우저에 반환됩니다.
5. 결과는 브라우저 Blob URL로 유지되며 사용자가 직접 다운로드합니다.

ViewGrid 코드 자체는 API 키나 이미지를 데이터베이스·오브젝트 스토리지에 저장하지 않습니다. 다만 Vercel과 선택한 생성 공급자는 각자의 인프라와 약관에 따라 요청을 처리하므로, 사용자는 해당 서비스의 데이터 처리 조건을 함께 확인해야 합니다.

## 브라우저 저장소

- `sessionStorage`: 카메라, 가이드 사용 여부, 일관성 옵션 등 비민감 설정만 현재 탭에 저장합니다.
- 저장하지 않음: API 키, 원본 이미지, 가이드 이미지, 참조 이미지, 생성 이미지, Blob URL.
- 탭을 닫으면 메모리의 API 키와 생성 결과가 사라집니다.

## 로그와 분석

기본 프로젝트에는 사용자 행동 분석 도구가 포함되어 있지 않습니다. 애플리케이션 코드도 API 키와 이미지 본문을 로그로 출력하지 않습니다. Vercel의 플랫폼 수준 요청 메타데이터는 Vercel 계정 설정과 정책의 적용을 받습니다.

## Service Worker

PWA Service Worker는 앱 셸과 정적 자산만 캐시합니다. `/api/` 요청, API 응답, 사용자 업로드와 생성 Blob은 캐시하지 않습니다.

## 삭제

서버 영구 저장소를 사용하지 않으므로 ViewGrid 내부에 별도의 이미지 삭제 요청 절차는 없습니다. 브라우저에서 새 프로젝트를 시작하거나 탭을 닫으면 메모리 객체가 제거됩니다. 다운로드한 파일은 사용자의 기기에서 직접 관리합니다.

## 배포자가 변경해야 하는 항목

공개 인스턴스 운영자는 저장소 URL, 운영자 연락 경로, 도메인, 추가 분석 도구나 쿠키 사용 여부를 이 문서에 반영해야 합니다. 기본 프로젝트에 없는 분석·광고·모니터링 도구를 추가하면 처리 목적과 보존 기간을 별도로 고지해야 합니다.
